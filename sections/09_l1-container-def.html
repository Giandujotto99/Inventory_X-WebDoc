Container Definition

The UInventoryContainerDefinition is the asset (UDataAsset) responsible for defining the structure, geometry, and access rules of a container. Thanks to its reusable nature, a single definition can be used to instantiate both the Root Containers of an Actor (such as the main backpack or the player's equipment slots) and the Child Containers dynamically created by specific items (such as a bag inside a backpack or the accessory slots of a weapon).

Container Types

The system adopts a unified model that supports two mutually exclusive container types:

Grid-Based (NxM Grid): Defines a flat geometric space where items are placed using coordinates and rotation (e.g., a 10x6 grid).

Slot-Based (Typed Slots): Defines a set of specific slots (e.g., Head slot, Torso slot, Weapon slot), each governed by its own rule profile.

Main Asset Properties

Key configurations within the asset include:

Structural Configuration:

ContainerTypeTag: Tag that identifies the nature of the container (e.g., InventoryX.Container.Type.Grid).

ContainerDimensions: Grid dimensions (Width and Height) expressed as FIntPoint.

SlotProfiles: List of references to UInventorySlotProfileDefinition for slot-based containers.

bAllowNestedContainers: Flag that enables or disables the ability to place containers inside other containers (Nesting).

bAllowRotation: Determines whether the container allows NxM items to be rotated.

Acceptance Filters (Tag-Driven):

AcceptedItemQuery: An FGameplayTagQuery that defines which items are allowed.

DeniedItemQuery: A query used to exclude specific categories of items. Note: Denial always takes priority over acceptance.

Access Policy:

AccessPolicyTags: Tags that define interaction permissions (e.g., InventoryX.Permission.ReadOnly for view-only containers or InventoryX.Permission.OwnerOnly to restrict access to the owner only).

Editor Validation

To prevent geometric or logical bugs, the asset implements automatic checks during saving:

Exclusivity: A container cannot be both grid and slot at the same time; if slot profiles are defined, the grid dimensions must be zero.

Grid Integrity: If the container is grid-based, both dimensions must be greater than 0.

Tag Consistency: The ContainerTypeTag must be present in the ContainerTags list.

Nesting Safe: If rotation is enabled but the container is slot-based, the system emits a warning since rotation has no geometric meaning for typed slots.

Relationship with Runtime

At runtime, the definition is used by the UInventoryComponent to generate an FInventoryContainerRecord. While the definition is immutable and shared, the runtime record maintains the instance-specific state, including dynamic tags and references to the items actually contained.
