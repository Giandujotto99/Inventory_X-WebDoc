Item Cosmetic

The UInventoryItemCosmetic asset represents the visual and descriptive pillar of Inventory_X. Its main function is to completely separate aesthetic and presentation data from the inventory simulation logic, ensuring that the core of the plugin remains lightweight and "blind" to heavy multimedia content.

Vision: Separation between Logic and Visual

In an enterprise multiplayer context, the dedicated server does not need to load textures, meshes, or audio files to validate the movement of an item. For this reason, all data that does not influence gameplay simulation (such as the localized name, icon, or 3D model) is isolated in this dedicated asset.

Integration and Isolation of the Dedicated Server

The UInventoryItemDefinition (the logical asset) points to its cosmetic counterpart via a TSoftObjectPtr.

Total isolation: The core runtime module (Inventory_X) never directly references the cosmetic class nor attempts to load it.

Safe access: The item definition exposes a getter method that returns the "soft" pointer without forcing it to load, leaving memory management responsibility exclusively to the client or the UI.

Asset Loading (Synchronous vs Asynchronous)

To avoid slowdowns (hitches) on the main thread when opening inventories dense with items, the plugin enforces strict loading guidelines:

Asynchronous Loading (Recommended): For production UI, the AsyncLoadItemCosmetic method provided by the client library is used. This loads the asset in the background and invokes a callback only when loading is complete.

Synchronous Loading: The LoadItemCosmetic function is available but its use is limited to rapid prototyping, Editor tools, or the sample module (Sample UI), where frame rate smoothness is not critical.

Typical Asset Properties

Within a UInventoryItemCosmetic, it is possible to configure:

Text Information: Display Name and description, natively supporting Unreal localization.

Iconography: 2D textures for the grid and icons for tooltips.

Mesh References: 3D models used for displaying the item in the world (pickup) or for character preview.

Effects: References to Sound Cue (SFX) for drag-and-drop or particle effects (VFX) tied to the item's rarity.

Packaging Optimization

Thanks to this architecture, developers can tag cosmetic assets (for example with a CosmeticOnly tag in the Asset Registry) to completely exclude them from the cook process of dedicated server builds, drastically reducing file sizes and server startup times.
