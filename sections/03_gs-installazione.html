Installation and Setup

The integration of Inventory_X is designed to be straightforward, whether you use a project based entirely on Blueprint or one based on C++. The plugin is structured into separate modules to ensure that the core remains lightweight and free of unnecessary dependencies.

1

Plugin Placement

To get started, copy the plugin folder into your project's Plugins directory (or install into the engine with FAB): 

YourProject/Plugins/Inventory_X/

Make sure that the Inventory_X.uplugin file is configured correctly. By default, the plugin is set as a Runtime module with a Default loading phase.

2

Module Configuration (C++ Only)

If your project uses C++, add the required modules to the YourProject.Build.cs file.

  • Layer 1 (Required): Add Inventory_X to your PublicDependencyModuleNames.
  • Layer 2 (Optional): If you need enterprise cross-inventory interactions, add Inventory_XLayer2.
  • Client Module (Optional): For using asynchronous cosmetic loading helpers, add Inventory_XClient.
3

Adding the Component

The heart of the system is the UInventoryComponent. To enable inventory logic on an Actor (such as your Character or Pawn):

  • Open the desired Actor in the Unreal Editor.
  • Click on Add Component and search for Inventory Component.
  • The component is marked as BlueprintSpawnableComponent, allowing you to easily add it to any Actor.
4

Multiplayer Configuration

Inventory_X is natively multiplayer-ready. To ensure proper network functionality:

  • The component is automatically set to replicate in the constructor (SetIsReplicatedByDefault(true)).
  • Make sure that the Actor that owns the component is also set to replicate.
5

Inventory Initialization

Once the component has been added, you need to configure the Root Containers (the main inventory spaces, such as the grid or equipment slots). This operation must occur only on the Server.

  • Via Blueprint: Use the BeginPlay event in the Actor and, after a HasAuthority check, use the Add Root Container node. You will need to provide a valid UInventoryContainerDefinition to define the size and rules of the grid or slots.
  • Initial Population: You can use the TryAddItem or TryEquipItem nodes to generate the player’s starting items at the moment of first spawn.
6

Gameplay Tags Configuration

The plugin uses Gameplay Tags for almost all of its logic. Upon installation, the system automatically registers the root namespaces (e.g., InventoryX.*).

💡 Suggestion

We recommend defining your specific tags (item categories, slot types) through Data Assets or the project tag tables to fully leverage the data-driven rules engine.
