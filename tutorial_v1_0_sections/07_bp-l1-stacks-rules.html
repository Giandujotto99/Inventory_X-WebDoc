Layer 1 - Stacks, Split, Merge, And Rules

Stacks are not just numbers in the UI. They are runtime items with quantity, merge rules, split limits, and server-side validation.

Stack Rule

Open DA_RuleStack_AmmoLight and DA_RuleStack_Medkit. Check these fields:

FieldEffect
MaxStackSizeMaximum quantity in a single runtime item.
bRequireMatchingFragmentsForMergeIf true, two stacks can merge only if their fragment data is compatible.
bAllowSplitAllows or denies creating a new stack through split.

Adding Quantity

When you call TryAutoAddItem with a quantity greater than 1, the system creates an item with that quantity if it does not exceed the max stack size.

Ammo Debug
Inventory.TryAutoAddItem(DA_Item_AmmoLight, BackpackContainer, 60)
Inventory.TryAutoAddItem(DA_Item_Medkit, BackpackContainer, 2)

If you request an invalid quantity or one above the limit, the result code can be InvalidQuantity.

Split Stack

To split a stack, you need the source item, quantity to separate, target container, and target placement.

TrySplitStack
On Split Stack Dialog Confirm
  SourceItem = selected item handle
  SplitQuantity = value from numeric input
  TargetContainer = backpack container handle
  TargetPlacement = grid placement selected by user
  Result = Inventory.TrySplitStack(SourceItem, SplitQuantity, TargetContainer, TargetPlacement)

Split UI

  1. Right-click a stackable item.
  2. Open a small WBP_SplitStackPopup widget.
  3. Show a range from 1 to Quantity - 1.
  4. When the user confirms, enter placement mode and then call TrySplitStack.

Merge Stack

When you drop a stack over another item of the same type, try merge first. If merge is not valid, you can try swap or move.

Drop On Item
On Drop SourceItem over TargetItem
  If SourceItem.DefinitionId == TargetItem.DefinitionId:
    Result = Inventory.TryMergeStack(SourceItemHandle, TargetItemHandle)
    If Result.ResultCode == Success:
      return
  Result = Inventory.TrySwapItems(SourceItemHandle, TargetItemHandle)

Placement Rules

Placement rules are tag queries applied to the container. For example, DA_RulePlacement_BackpackOnly prevents placing an item in a container that does not match InventoryX.Container.Type.PlayerBackpack.

To test the rule:

  1. Try adding DA_Item_Rifle directly into the equipment container with TryAddItem.
  2. The rule should deny the operation or the slot layout should reject a grid placement.
  3. Equipping should use TryEquipItem, not TryMoveItem toward an arbitrary slot.

Common Errors

ProblemLikely cause
Merge rejectedDifferent definition, full stack, or incompatible fragments.
Split rejectedQuantity lower than 1, equal to the total quantity, or bAllowSplit disabled.
Stack does not show quantityThe UI is reading the Data Asset instead of ItemView.Quantity.
Auto add failsContainer full, item too large, or placement rule does not match the container.

Verification

  • Ammo and medkit show quantity in the UI.
  • You can split a stack into a free cell.
  • You can merge two compatible stacks.
  • The UI shows a readable message when a rule denies the operation.
