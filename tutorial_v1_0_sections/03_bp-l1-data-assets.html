Layer 1 - Creating The Core Data Assets

In this tutorial you create containers, slots, rules, fragments, cosmetics, and item definitions from scratch. By the end, you will have the minimum assets for a backpack, equipment slots, a rifle, ammo, a medkit, and a nestable backpack.

How To Create A Data Asset

In the Content Browser, use Right Click > Miscellaneous > Data Asset, then choose the correct Inventory_X class. If you cannot find the classes, make sure the plugin is enabled and that you are searching inside the Data Asset class picker.

Containers

Asset Class Recommended values
DA_Container_PlayerBackpack InventoryContainerDefinition Type InventoryX.Container.Type.PlayerBackpack, Dimensions 8 x 6, Allow Rotation true, Allow Nested Containers true.
DA_Container_PlayerEquipment InventoryContainerDefinition Type InventoryX.Container.Type.Equipment, Dimensions 0 x 0, Slot Profiles: primary weapon and head, Allow Nested Containers false.
DA_Container_BackpackPocket InventoryContainerDefinition Type InventoryX.Container.Type.PlayerBackpack, Dimensions 4 x 4, Allow Rotation true.
Grid or slot?

A container is grid-based if it has non-zero dimensions and no slot profiles. It is slot-based if it has slot profiles. Do not configure both on the same container.

Slot Profiles

Asset Class Setup
DA_Slot_PrimaryWeapon InventorySlotProfileDefinition Slot Type Tag InventoryX.Slot.Equip.PrimaryWeapon. Accepted Item Query: item must match InventoryX.Item.Category.Weapon.Rifle. Max Items 1.
DA_Slot_Head InventorySlotProfileDefinition Slot Type Tag InventoryX.Slot.Equip.Head. Accepted Item Query: use a helmet tag when you add a helmet. Max Items 1.

Rules

Asset Class Setup
DA_RuleStack_AmmoLight InventoryStackRuleDefinition Max Stack Size 120, Allow Split true.
DA_RuleStack_Medkit InventoryStackRuleDefinition Max Stack Size 4, Allow Split true.
DA_RulePlacement_BackpackOnly InventoryPlacementRuleDefinition Required Container Query: container must match InventoryX.Container.Type.PlayerBackpack.
DA_RuleEquip_PrimaryWeapon InventoryEquipRuleDefinition Compatible Slot Query: slot must match InventoryX.Slot.Equip.PrimaryWeapon. Exclusive Slot true.

Fragments

Create these assets in Definitions/Fragments. Each fragment has a FragmentTypeTag and DefaultFragmentData.

Asset DefaultFragmentData Values
DA_Fragment_Ammo InventoryAmmoFragmentData Max Ammo 30, Current Ammo 30, Ammo Type Tag InventoryX.Item.Category.Ammo.Light.
DA_Fragment_Durability InventoryDurabilityFragmentData Max Durability 100, Current Durability 100.
DA_Fragment_Condition InventoryConditionFragmentData Condition Tag of your choice, for example InventoryX.State.Condition.Pristine if available.

Cosmetics

Create an InventoryItemCosmetic for every important item. For now, you can use icons and meshes from the Third Person Template or placeholders created inside the project.

  • DA_Cosmetic_Rifle: Display Name Rifle, short description, placeholder icon.
  • DA_Cosmetic_AmmoLight: Display Name Light Ammo.
  • DA_Cosmetic_Medkit: Display Name Medkit.
  • DA_Cosmetic_Backpack: Display Name Backpack.

Item Definitions

Asset Essential setup
DA_Item_Rifle Tags InventoryX.Item.Category.Weapon.Rifle, Shape 2 x 5, Supports Rotation true, backpack-only Placement Rule, primary weapon Equip Rule, ammo and durability fragments, rifle Cosmetic.
DA_Item_AmmoLight Tags InventoryX.Item.Category.Ammo.Light, Shape 1 x 1, ammo Stack Rule, backpack-only Placement Rule, ammo Cosmetic.
DA_Item_Medkit Tags InventoryX.Item.Category.Consumable.Medkit, Shape 2 x 2, medkit Stack Rule, backpack-only Placement Rule, medkit Cosmetic.
DA_Item_Backpack Tags InventoryX.Item.Category.Equipment.Backpack, Shape 2 x 3, Child Container Definitions: DA_Container_BackpackPocket, backpack Cosmetic.
Save your assets

Save all assets after configuring them. Some validation and automatic tags are applied on save, for example the capability tag on items with child containers.

Verification

  • DA_Container_PlayerBackpack should be grid-based.
  • DA_Container_PlayerEquipment should be slot-based.
  • DA_Item_Rifle should be equippable.
  • DA_Item_AmmoLight should be stackable.
  • DA_Item_Backpack should have child containers.
