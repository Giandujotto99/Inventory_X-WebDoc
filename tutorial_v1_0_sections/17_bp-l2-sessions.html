Layer 2 - Opening And Controlling Sessions

Before manipulating an external inventory, you must open a session. In this tutorial, you wire trace, async open, close, resume, upgrade, and downgrade.

Trace Toward The Provider

In the player controller or character, create a function named FindInteractionProvider.

Find Provider
Line Trace from camera
If HitActor has InventoryExternalInventoryProviderComponent:
  return HitActor
Else:
  return None

Open Interaction Session Async

Use the async node to get clean Blueprint callbacks.

Open Chest
Input Interact
  ProviderActor = FindInteractionProvider
  OpenInteractionSessionAsync
    InteractionComponent = InventoryInteraction
    ProviderActor = ProviderActor
    RequestedMode = WriteExclusive
    bAllowReadOnlyFallback = true
    ContextTags = empty or InventoryX.Interaction.Context.PlayerExternal

  OnOpened:
    Store SessionView
    Create WBP_InteractionRoot
    Add To Viewport

  OnRejected:
    Print Result.ResultCode

Session View

FInventoryInteractionSessionView is the main source for Layer 2 UI.

  • bHasActiveSession
  • SessionId and SessionEpoch
  • SessionState
  • SessionMode
  • LockState
  • CapabilitySnapshot
  • PrimaryEndpoint and SecondaryEndpoint
  • bPredictionActive and PendingOperationCount

Session Controls

For close, resume, upgrade, and downgrade, you can use direct functions or the RequestInteractionControlAsync node.

ActionControl TypeUse
Close chestCloseSessionWhen you close the UI or intentionally stop interacting.
ResumeRequestResumeWhen a suspended session becomes valid again.
UpgradeRequestWriteLockUpgradeMove from read-only to write-exclusive.
DowngradeDowngradeToReadOnlyRelease write lock while keeping the session open.

Events To Bind

  • OnInteractionSessionOpened
  • OnInteractionSessionClosed
  • OnInteractionSessionSuspended
  • OnInteractionSessionResumed
  • OnInteractionLockStateChanged

Verification

  • Interacting with the chest opens a session.
  • The UI shows session state, mode, and lock.
  • Close session closes the UI and releases the lock.
  • Moving away changes the session to suspended if the policy requires it.
