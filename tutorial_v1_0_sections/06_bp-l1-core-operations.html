Layer 1 - Main Inventory Operations

Now you make the UI interactive: add, move, swap, rotate, remove, and preview placement. The goal is to connect UI input to command wrappers, not manually edit widgets and arrays.

Placement Info In Blueprint

Many operations require FInventoryPlacementInfo. In Blueprint, use the Make InventoryPlacementInfo node.

PlacementFieldsUse
GridKind = Grid, GridTopLeft = X/Y, RotationBackpacks, chests, bags, NxM containers.
SlotKind = Slot, SlotIndexEquipment slots.
NoneKind = NoneUsed when you want the system to resolve a destination, for example in some unequip cases.

Adding Items For Debug

Add debug buttons in the UI or temporary inputs on the character.

TryAutoAddItem
Button Add Medkit
  Inventory.TryAutoAddItem(DA_Item_Medkit, BackpackContainer, 1)
  Branch WasOperationSuccessful(Result)
    False: Print GetResultCodeDisplayName(Result.ResultCode)

For the final UI, do not depend on BackpackContainer saved server-side. Find the container through GetRootContainerViews and choose the grid-based one or the one with the correct tag.

Preview Before Drop

When you drag an item over a cell, do not call TryMoveItem immediately. Use a preview first.

Hover Preview
On Item Drag Over Cell
  Make InventoryPlacementInfo
    Kind = Grid
    GridTopLeft = Cell X/Y
    Rotation = CurrentDragRotation
  Inventory.CanPlaceItemAtPlacement(TargetContainer, Placement, DraggedItemDefinition, true, DraggedItemHandle)
  If bSuccess:
    Highlight cells green
  Else:
    Highlight cells red and show FailReason

Move Item

When the item is dropped on a valid cell:

TryMoveItem
On Drop Item On Cell
  Placement = Make InventoryPlacementInfo(Grid, Cell X/Y, DragRotation)
  Result = Inventory.TryMoveItem(DraggedItemHandle, TargetContainerHandle, Placement)
  If Result.Status == Pending:
    Mark item as pending in UI
  If Result.ResultCode != Success:
    Show error feedback

Swap Items

If you drop an item over another item and you do not want to merge, use TrySwapItems.

  • Source: handle of the dragged item.
  • Target: handle of the item under the mouse.
  • The system checks space, rules, and collisions.

Rotate Item

During drag, you can use an input such as R to change the visual rotation. When you want to apply rotation to an item already in the inventory, use TryRotateItem.

Rotation requires three permissions

The item must have bSupportsRotation, the container must have bAllowRotation, and the placement rule must not deny rotation.

Remove Item

To delete an item without spawning it in the world, use TryRemoveItem. For physical world drops, you will use TryDropItem in the dedicated tutorial.

Result Codes To Handle

Result CodeUI meaning
SuccessOperation accepted or completed.
ContainerFullNo valid space is available.
CellOccupiedThe selected position collides with another item.
OutOfBoundsThe item goes outside the grid bounds.
InvalidRotationRotation is not allowed.
RuleDeniedA rule or tag query blocked the operation.
AccessDeniedYou are calling from an unauthorized role or on a protected container.

Verification

  • You can drag an item from one cell to another.
  • The preview turns red if the position is invalid.
  • Swap works only when both destinations are valid.
  • The UI never modifies inventory state directly: it always calls command wrappers.
