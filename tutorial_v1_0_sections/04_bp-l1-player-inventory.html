Layer 1 - Initializing The Player Inventory

Now you connect the assets to the Third Person Character and create the runtime root containers: backpack grid and equipment slots.

Preparing The Character

1

Open the Third Person Character

Duplicate the template character into /Game/Tutorials/InventoryX/Blueprints/Characters/BP_InventoryCharacter or use your existing project character.

2

Add InventoryComponent

In the Components panel, add InventoryComponent. Rename it to Inventory.

3

Enable replication on the actor

In the character, make sure Replicates = true. The component replicates inventory data, but the owner actor must correctly participate in networking.

Server-Side BeginPlay

AddRootContainer only works on Authority. If you call it on the client, it returns an invalid handle.

Blueprint Flow
Event BeginPlay
  Branch: Has Authority
    True:
      Inventory - AddRootContainer(BackpackDefinition, "PlayerBackpack") -> Set BackpackContainer
      Inventory - AddRootContainer(EquipmentDefinition, "PlayerEquipment") -> Set EquipmentContainer
      Inventory - TryAutoAddItem(DA_Item_Rifle, BackpackContainer, 1)
      Inventory - TryAutoAddItem(DA_Item_AmmoLight, BackpackContainer, 60)
      Inventory - TryAutoAddItem(DA_Item_Medkit, BackpackContainer, 2)
Do not rely only on local variables

In multiplayer, handles saved in Blueprint variables on the server are not automatically a reliable source for the client UI. The UI should use GetRootContainerViews or GetRootContainers on the replicated component.

Binding Events

In the character, create a function named RefreshInventoryDebug. Then bind it to component events.

Event Binding
Event BeginPlay
  Inventory - Bind Event to OnInventoryChanged
  Inventory - Bind Event to OnOperationCompleted
  Inventory - Bind Event to OnOperationFailed

Inside RefreshInventoryDebug, use GetRootContainerViews, print how many root containers exist, and how many items each container contains.

Reading Root Containers

To test without a complete UI:

  1. Call GetRootContainerViews from the Inventory component.
  2. Use a ForEachLoop.
  3. For each ContainerView, print Dimensions, bIsGridBased, bIsSlotBased, and the number of Items.

PIE Verification

  • In single player, you should see two root containers: backpack and equipment.
  • The backpack should contain rifle, ammo, and medkit if there is enough space.
  • If TryAutoAddItem fails, check Asset Manager, tags, placement rule, and container dimensions.
