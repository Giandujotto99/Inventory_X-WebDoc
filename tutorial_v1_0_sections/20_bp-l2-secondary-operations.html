Layer 2 - Operations Inside The Secondary Inventory

Besides transferring items between player and chest, Layer 2 allows you to manipulate the external inventory directly, if the policy grants the required capabilities.

When They Are Needed

  • Reorder items inside a chest without moving them to the player.
  • Rotate items inside the stash.
  • Split or merge stacks in the external container.
  • Manage equipment inside a special external inventory, if your game requires it.

Move Inside Secondary

TrySecondaryMoveItem
SourceItem = MakeScopedItemHandle(Secondary, ItemHandle)
SourceContainer = MakeScopedContainerHandle(Secondary, ParentContainerHandle)
TargetContainer = MakeScopedContainerHandle(Secondary, TargetContainerHandle)
TargetPlacement = Make InventoryPlacementInfo(Grid, Cell X/Y, Rotation)

Preview = Interaction.PreviewSecondaryMoveItem(SourceItem, SourceContainer, TargetContainer, TargetPlacement)
If Preview.bWouldSucceed:
  Interaction.TrySecondaryMoveItem(SourceItem, SourceContainer, TargetContainer, TargetPlacement)

Equip And Unequip Inside Secondary

These operations are useful only if the secondary inventory has an equipment container. Examples: armory, mannequin, vehicle loadout, vendor equip preview.

  • TrySecondaryEquipItem: equips an item into an equipment container on the secondary endpoint.
  • TrySecondaryUnequipItem: removes an equipped item on the secondary endpoint.

UI

When the user drags an item inside the secondary panel, choose the operation in this order:

  1. If the target is an empty cell: TrySecondaryMoveItem.
  2. If the target is a compatible stack item: TrySecondaryMergeStack.
  3. If the target is an incompatible item: TrySecondarySwapItems.
  4. If the input is rotate: TrySecondaryRotateItem.

Verification

  • You can reorder items in the chest without transferring them to the player.
  • If the policy disables a capability, the UI blocks or explains the action.
  • Operations update the fused view when they are predicted.
