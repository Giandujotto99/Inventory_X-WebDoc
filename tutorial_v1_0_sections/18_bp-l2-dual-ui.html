Layer 2 - Two-Inventory UI

The Layer 2 UI shows the player inventory and the external inventory on the same screen. It must respect session state, lock, and capabilities.

Widgets To Create

WidgetResponsibility
WBP_InteractionRootRoot UI for the Layer 2 session.
WBP_InventoryEndpointPanelDisplays an endpoint: primary or secondary.
WBP_SessionStatusBarShows mode, lock, state, pending operations, and result feedback.
WBP_CapabilityHintDisables or explains actions not granted by the policy.

Root Variables

  • InteractionComponent: InventoryInteractionComponent Object Reference, Expose on Spawn.
  • SessionView: InventoryInteractionSessionView.
  • PrimaryPanel: reference to WBP_InventoryEndpointPanel.
  • SecondaryPanel: reference to WBP_InventoryEndpointPanel.

Session Refresh

WBP_InteractionRoot.Refresh
SessionView = InteractionComponent.GetActiveSessionView()
StatusBar.SetSessionView(SessionView)

PrimaryInventory = GetEndpointInventoryComponent(SessionView, Primary)
SecondaryInventory = GetEndpointInventoryComponent(SessionView, Secondary)

PrimaryPanel.SetInventoryComponent(PrimaryInventory, Primary)
SecondaryPanel.SetInventoryComponent(SecondaryInventory, Secondary)

Secondary Fused View

For the secondary endpoint, when prediction is active, use GetSecondaryPredictedView. This lets you display authoritative state plus predicted changes that have not been confirmed yet.

Secondary Panel
If EndpointRole == Secondary:
  FusedView = InteractionComponent.GetSecondaryPredictedView()
  If FusedView.bIsValid:
    Draw FusedView.RootContainers
  Else:
    Draw SecondaryInventory.GetRootContainerViews()

Capability-Based UI

Do not show active buttons if the session does not have the required capability. Use HasCapability from the Layer 2 Blueprint Library.

UI actionRequired capability
Deposit from player to chestDepositIntoExternalInventory
Withdraw from chest to playerWithdrawFromExternalInventory
Move item inside chestMoveWithinExternalInventory
Rotate item inside chestRotateWithinExternalInventory
Upgrade write lockUpgradeToWriteLock
Resume suspended sessionResumeSuspendedSession

Refresh On Events

Bind these events on the root widget:

  • OnInteractionSessionOpened
  • OnInteractionSessionClosed
  • OnInteractionSessionSuspended
  • OnInteractionSessionResumed
  • OnInteractionLockStateChanged
  • OnInteractionOperationPredicted
  • OnInteractionOperationConfirmed
  • OnInteractionOperationRejected
  • OnInteractionOperationCorrected

Verification

  • Opening the chest shows two panels: player and chest.
  • The status bar shows read-only or write-exclusive.
  • If you do not have write lock, mutating controls are disabled.
  • The secondary panel uses fused view when there are pending operations.
