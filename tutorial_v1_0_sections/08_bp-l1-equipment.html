Layer 1 - Equipment System

Equipment in Inventory_X is a slot-based container. Items are equipped through TryEquipItem and removed through TryUnequipItem.

Check The Assets

  • DA_Container_PlayerEquipment must have SlotProfiles configured.
  • DA_Slot_PrimaryWeapon must accept items with the InventoryX.Item.Category.Weapon.Rifle tag.
  • DA_Item_Rifle must have DA_RuleEquip_PrimaryWeapon assigned.
  • DA_RuleEquip_PrimaryWeapon must have a Compatible Slot Query compatible with InventoryX.Slot.Equip.PrimaryWeapon.

Drawing Slots

In WBP_EquipmentPanel, receive a slot-based ContainerView. Every item inside that container has Placement.Kind = Slot and Placement.SlotIndex.

Equipment Refresh
SetContainerView(ContainerView)
  Clear equipment slot widgets
  For each SlotProfile in ContainerView.ContainerDefinition.SlotProfiles:
    Create WBP_EquipmentSlot with SlotIndex
  For each ItemView in ContainerView.Items:
    Find widget by ItemView.Placement.SlotIndex
    Set ItemView on that slot

Equip From UI

When you drop an item on the primary weapon slot:

TryEquipItem
On Drop Item On EquipmentSlot
  Preview = Inventory.PreviewEquipItem(ItemHandle, EquipmentContainerHandle, SlotIndex)
  If Preview.bWouldSucceed:
    Result = Inventory.TryEquipItem(ItemHandle, EquipmentContainerHandle, SlotIndex)
  Else:
    Show Preview.DebugMessage or ResultCode

Unequip

To remove an item from a slot, you can specify a destination or leave container and placement invalid so the system can resolve a destination automatically when possible.

TryUnequipItem
On Unequip Button
  Preview = Inventory.PreviewUnequipItem(ItemHandle, InvalidContainerHandle, NonePlacement)
  If Preview.bWouldSucceed:
    Inventory.TryUnequipItem(ItemHandle, Preview.ResolvedTargetContainer, Preview.ResolvedTargetPlacement)

Tags Applied On Equip

InventoryEquipRuleDefinition can apply tags to the owner when the item is equipped. This is useful for enabling abilities, animations, input, or UI state.

Example: add a tag such as InventoryX.State.Equipped.PrimaryWeapon to TagsAppliedOnEquip in the rule, then check owner tags or react to inventory events in the character.

Common Errors

ErrorSolution
RuleDenied during equipCheck Compatible Slot Query and the slot Accepted Item Query.
InvalidHandleYou are using an old item handle. Read the ItemView again after each refresh.
Item equipped but UI does not changeThe UI is not refreshing on OnInventoryChanged or is reading the wrong container.
Unequip failsNo valid destination exists in the backpack or the item is too large for the available space.

Verification

  • Dropping the rifle on the primary weapon slot moves it into the equipment container.
  • The slot rejects incompatible items, such as medkits.
  • Unequip moves the item back into the backpack if there is enough space.
