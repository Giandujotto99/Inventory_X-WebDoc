Blueprint Track - Complete Setup From Scratch

This series builds a complete Blueprint inventory system starting from the Third Person Template. It does not use the plugin sample modules and does not use assets from the plugin Content folder.

UI note

The plugin does not replace the need to create a UI. This tutorial track does not cover widget implementation in every small detail; instead, it explains how and where to use the Inventory_X functions. For a deeper reference, the Unreal Engine 5.7 version of Inventory_X includes 3 complete sample maps with widgets: 2 for C++ and one very detailed Blueprint sample where you can inspect all widgets and setups.

Track rule

All tutorial assets must live inside the project Content folder, for example under /Game/Tutorials/InventoryX. If plugin content is visible in the Content Browser, do not use it as the base for this track.

Prerequisites

  • Unreal Engine project created from the Third Person Template.
  • Inventory_X plugin already enabled.
  • Basic knowledge of Blueprints, Widget Blueprints, Data Assets, and Gameplay Tags.
  • For multiplayer chapters: at least two PIE players or a local Dedicated Server setup.

Asset Manager

Inventory_X identifies items through FInventoryDefinitionId, which is based on Primary Assets. For this reason, the Asset Manager must know where to find InventoryItemDefinition assets.

1

Open Project Settings

Go to Project Settings > Asset Manager.

2

Add a Primary Asset Type

Add an entry with Primary Asset Type = InventoryItemDefinition and Asset Base Class = InventoryItemDefinition.

3

Set the directory

Add /Game/Tutorials/InventoryX/Definitions/Items to the directories. Leave Has Blueprint Classes disabled and use Cook Rule = Always Cook.

Common mistake

If you skip the Asset Manager setup, items may appear to work in the editor but fail in multiplayer or packaged builds because the server cannot resolve the DefinitionId.
