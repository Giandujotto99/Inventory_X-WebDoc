Layer 1 - Complete System Checklist

This page closes the Layer 1 path. Use it as a checklist before moving to Layer 2.

Features You Should Have

AreaCompleted when...
SetupAsset Manager, tags, folders, and tutorial map are configured.
DefinitionsYou have items, containers, slots, rules, fragments, and cosmetics created inside the project.
Player inventoryThe character creates root containers server-side and replicates them to clients.
UIThe UI reads only from views and refreshes on OnInventoryChanged.
OperationsAdd, move, swap, rotate, and remove work with result feedback.
StacksSplit and merge respect max stack size and fragments.
EquipmentEquip and unequip respect slot profile and equip rule.
FragmentsTooltips and UI read ammo, durability, and condition.
NestingItems with child containers open a nested panel.
Drop/PickupItems can leave the inventory and re-enter from the world.
PredictionThe UI handles pending, confirmed, rejected, and corrected operations.

Standard Debug Flow

When something does not work, always check in the same order:

  1. Is the Data Asset configured correctly?
  2. Does the tag query match the runtime tags?
  3. Is the container grid-based or slot-based as expected?
  4. Are you calling the node on the correct network role?
  5. Is the result code telling you the real reason?
  6. Is the UI reading updated views or old variables?

Ready For Layer 2

Move to Layer 2 once the player inventory is stable. Layer 2 does not replace Layer 1: it uses it to coordinate interactions between two inventory components, such as player and chest.
