Layer 2 - Operations Between Player And External Container

In this tutorial, you wire transfer, split, merge, and swap between primary and secondary endpoints using scoped handles and previews.

Create Scoped Handles

Before calling a Layer 2 operation, convert item handles and container handles into scoped handles.

Scoped Handles
SourceScopedItem = MakeScopedItemHandle(SourceEndpointRole, SourceItemView.ItemHandle)
SourceScopedContainer = MakeScopedContainerHandle(SourceEndpointRole, SourceItemView.ParentContainerHandle)
TargetScopedContainer = MakeScopedContainerHandle(TargetEndpointRole, TargetContainerView.ContainerHandle)

Transfer Item

Use transfer when moving a whole item between player and chest.

TryTransferItem
On Drop Item From Primary To Secondary Cell
  SourceItem = MakeScopedItemHandle(Primary, ItemHandle)
  SourceContainer = MakeScopedContainerHandle(Primary, ParentContainerHandle)
  TargetContainer = MakeScopedContainerHandle(Secondary, ChestContainerHandle)
  TargetPlacement = Make InventoryPlacementInfo(Grid, Cell X/Y, Rotation)

  Preview = Interaction.PreviewTransferItem(SourceItem, SourceContainer, TargetContainer, TargetPlacement)
  If Preview.bWouldSucceed:
    Result = Interaction.TryTransferItem(SourceItem, SourceContainer, TargetContainer, TargetPlacement)

Split Across Endpoints

Use Layer 2 TrySplitStack when you want to transfer only part of a stack.

Cross Split
Interaction.TrySplitStack(
  SourceItem,
  SourceContainer,
  SplitQuantity,
  TargetContainer,
  TargetPlacement)

Merge Across Endpoints

When dropping a player stack over a stack in the chest, try cross-endpoint merge first.

Cross Merge
SourceItem = MakeScopedItemHandle(Primary, DraggedItemHandle)
SourceContainer = MakeScopedContainerHandle(Primary, DraggedParentContainer)
TargetItem = MakeScopedItemHandle(Secondary, TargetItemHandle)

Preview = Interaction.PreviewMergeStack(SourceItem, SourceContainer, TargetItem)
If Preview.bWouldSucceed:
  Interaction.TryMergeStack(SourceItem, SourceContainer, TargetItem)

Swap Across Endpoints

If merge is not valid, you can try a swap between player and chest.

Cross Swap
Interaction.TrySwapItems(SourceItem, SourceContainer, TargetItem)

Result Feedback

FInventoryInteractionPreviewResult and FInventoryInteractionSubmitResult include both the Layer 2 result and the Layer 1 results for primary and secondary. If an operation fails, show which level failed.

Verification

  • You can deposit an item from the player to the chest.
  • You can withdraw an item from the chest to the player.
  • Split stack creates only the requested quantity in the other endpoint.
  • Merge and swap respect rules and capabilities.
