Layer 2 - PlayerController And External Provider

Layer 2 starts with two setups: the PlayerController must have an InventoryInteractionComponent, and the chest must expose an InventoryComponent through a provider.

PlayerController

1

Create BP_InventoryPlayerController

Create a PlayerController Blueprint in Blueprints/Controllers.

2

Add InventoryInteractionComponent

In the Components panel, add InventoryInteractionComponent and name it InventoryInteraction.

3

Configure GameMode

Set your GameMode to use BP_InventoryPlayerController.

Primary inventory resolution

InventoryInteractionComponent searches for the primary inventory on the PlayerController and on the Pawn. If it finds more than one InventoryComponent, set ExplicitPrimaryInventoryComponent to avoid ambiguity.

Chest Actor

Create BP_WorldChest in Actors/WorldInventories.

  • Parent class: Actor.
  • Add a mesh or cube placeholder.
  • Add a SceneComponent named InteractionAnchor.
  • Add an InventoryComponent named ChestInventory.
  • Add an InventoryExternalInventoryProviderComponent named ExternalProvider.
  • Set actor Replicates = true.

Provider Component

On the ExternalProvider component, set:

FieldValue
ProvidedInventoryComponentChestInventory
InteractionAnchorComponentInteractionAnchor
PolicyDefinitionYou will create it in the next tutorial. For now, leave it empty or assign it later.
ProviderTagsInventoryX.Container.Type.WorldContainer or a specific chest tag.
ProviderDebugNameWorldChest

Initializing The Chest Inventory

In the chest BeginPlay, on Authority, create a root container and add some items.

BP_WorldChest BeginPlay
Event BeginPlay
  Branch Has Authority
    True:
      ChestInventory.AddRootContainer(DA_Container_WorldChest, "ChestGrid") -> ChestContainer
      ChestInventory.TryAutoAddItem(DA_Item_AmmoLight, ChestContainer, 40)
      ChestInventory.TryAutoAddItem(DA_Item_Medkit, ChestContainer, 1)

Container Definition For The Chest

If you have not created it yet, add DA_Container_WorldChest:

  • Class InventoryContainerDefinition.
  • Container Type Tag InventoryX.Container.Type.WorldContainer.
  • Dimensions 8 x 4.
  • Allow Rotation true.

Verification

  • The PlayerController has exactly one valid InventoryInteractionComponent.
  • The chest replicates.
  • The provider returns the correct ChestInventory.
  • The chest creates its root container only on the server.
