Layer 2 - Overview

Layer 2 is used when a player interacts with an external inventory: chest, stash, vendor, corpse loot, vehicle cargo, or any actor that exposes an InventoryComponent.

When To Use Layer 1 And Layer 2

ScenarioCorrect layer
Move items inside the player backpackLayer 1
Equip a player itemLayer 1
Open a chest and transfer itemsLayer 2
Two players inspect the same stashLayer 2
Permissions, locks, range, and line of sight on an external providerLayer 2

Primary And Secondary

Layer 2 always works with two endpoints:

  • Primary endpoint: the inventory of the player currently interacting.
  • Secondary endpoint: the external inventory exposed by the provider, for example a chest.

Session

Before transferring items, you must open a session. The session contains:

  • SessionId and SessionEpoch.
  • State: opening, active read-only, active write-exclusive, suspended, closing.
  • Lock state.
  • Capabilities granted by the policy.
  • References to primary and secondary endpoints.

Scoped Handles

In Layer 1, an item handle identifies an item inside a single InventoryComponent. In Layer 2, you also need to specify which endpoint the handle belongs to.

Scoped Handle
FInventoryScopedItemHandle
  EndpointRole = Primary or Secondary
  ItemHandle = item handle inside that InventoryComponent

FInventoryScopedContainerHandle
  EndpointRole = Primary or Secondary
  ContainerHandle = container handle inside that InventoryComponent

Layer 2 Tutorial Goal

You will build a complete multiplayer chest with:

  • External provider with policy.
  • Read-only and write-exclusive sessions.
  • Two-panel UI.
  • Transfer, split, merge, and swap between player and chest.
  • Operations inside the secondary inventory.
  • Locks and two-player tests.
  • Prediction and fused view.
