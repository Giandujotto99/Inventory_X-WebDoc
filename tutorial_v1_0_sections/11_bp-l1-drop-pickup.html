Layer 1 - World Drop And Pickup

Now you turn an inventory item into a world actor and pick it up again, preserving quantity, fragments, and child containers when possible.

Create The Pickup Actor

Create BP_InventoryPickup in Actors/Pickups.

  • Parent class: Actor.
  • Add a StaticMeshComponent as a visual placeholder.
  • Add PickupableItemComponent and name it PickupableItem.
  • Set actor Replicates = true.
  • Implement the InventoryDropPickupActor interface.
  • In the GetPickupableItemComponent function, return the PickupableItem component.

Configure ItemToDrop

Open the item definitions you want to drop, for example DA_Item_Rifle, DA_Item_Medkit, and DA_Item_AmmoLight. Assign BP_InventoryPickup to the ItemToDrop field.

Drop Transform Provider

TryDropItem requires the owner of the InventoryComponent to implement InventoryDropTransformProvider. Implement the interface on your character and fill in CalculateDropTransform.

CalculateDropTransform
Input: SourceInventory, ItemView, ItemDefinition, RequestingController
Get Actor Location
Get Control Rotation or Actor Forward Vector
OutTransform Location = ActorLocation + ForwardVector * 120 + UpVector * 30
OutTransform Rotation = ActorRotation
Return true

Call TryDropItem

From the item context menu or a debug button:

Drop
On Drop Item Button
  Result = Inventory.TryDropItem(SelectedItemHandle)
  If Result.ResultCode != Success:
    Show GetResultCodeDisplayName(Result.ResultCode)

Picking Up A Pickup

To pick up, trace from the player into the world and look for an actor with PickupableItemComponent or the InventoryDropPickupActor interface.

Pickup
Input Interact
  Line Trace
  If HitActor implements InventoryDropPickupActor:
    Server_RequestPickup(HitActor)

Server_RequestPickup (Run On Server)
  Inventory.TryAutoPickupItem(HitActor, BackpackContainer)
Server-side pickup

For this version, treat pickup as server-side. If the input starts on the client, create a Run On Server Custom Event that calls TryPickupItem or TryAutoPickupItem.

Presentation Payload

PickupableItemComponent exposes useful data for world-space UI:

  • GetPresentationPayload: item definition, quantity, fragments, child container item counts.
  • OnItemDropped: called when the pickup is ready.
  • OnItemPickedUp: useful for effects and sounds.

Verification

  • Dropping an item spawns BP_InventoryPickup.
  • The item disappears from the inventory.
  • Picking up the pickup returns the item to the backpack.
  • If the backpack is full, the pickup remains in the world and the operation fails with a coherent result.
