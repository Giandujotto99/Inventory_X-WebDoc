Layer 1 - Fragments And Item State

Fragments let you add modular data to items without creating a different item class for every case. In Blueprints, you will mainly use them to read ammo, durability, and condition in UI or gameplay logic.

Fragment Definition

Each InventoryItemFragmentDefinition contains:

  • FragmentTypeTag: identifies the fragment type.
  • DefaultFragmentData: initial data created on each runtime item.
  • ContributedRuntimeTags: tags added to the item when the fragment is present.

Assigning Fragments To Items

Open DA_Item_Rifle and assign:

  • DA_Fragment_Ammo
  • DA_Fragment_Durability

Open DA_Item_Medkit and optionally assign DA_Fragment_Condition if you want to test a state.

Reading Fragments In Blueprint

From the Inventory component, you can read an item's fragments through dedicated nodes.

NodeUse
GetItemFragmentsReturns all fragment views on the item.
GetFragmentByTagReturns a specific fragment using the FragmentTypeTag.
HasFragmentChecks whether the item has a specific fragment.
TryGetAmmoFragmentReturns InventoryAmmoFragmentData.
TryGetDurabilityFragmentReturns InventoryDurabilityFragmentData.
TryGetConditionFragmentReturns InventoryConditionFragmentData.

Tooltip With Ammo And Durability

WBP_ItemTooltip
SetItemView(ItemView)
  Inventory.TryGetAmmoFragment(ItemView.ItemHandle, OutAmmo)
    True: Show CurrentAmmo / MaxAmmo and AmmoTypeTag
  Inventory.TryGetDurabilityFragment(ItemView.ItemHandle, OutDurability)
    True: Show CurrentDurability / MaxDurability
  Inventory.TryGetConditionFragment(ItemView.ItemHandle, OutCondition)
    True: Show ConditionTag

Runtime Tags From Fragments

If a fragment contributes runtime tags, they appear in ItemView.RuntimeTags. You can use them to:

  • Color an item in UI.
  • Filter items with QueryItemsByTag.
  • Allow or deny placement in specific containers.
  • Show states such as loaded, damaged, broken, or cursed.
Modifying fragment data

The public Blueprint surface of Layer 1 exposes fragment queries and reading. If your game needs to change ammo or durability as real runtime logic, create your own gameplay layer that invokes safe operations or extend the plugin with dedicated APIs. Do not modify only the UI.

Verification

  • The rifle tooltip shows ammo and durability.
  • Items without fragments do not show empty sections.
  • Fragment runtime tags are visible in the debug UI.
