Layer 1 - Nested Containers

An item can contain other containers. This enables backpacks, pouches, weapon cases, and objects with internal inventory space.

Backpack Item Setup

Open DA_Item_Backpack and check:

  • ChildContainerDefinitions contains DA_Container_BackpackPocket.
  • The backpack shape size is large enough, for example 2 x 3.
  • The child container is grid-based, for example 4 x 4.
  • The placement rule allows the backpack to be placed in the player backpack.

Adding The Backpack

For testing, add the backpack to the player inventory with TryAutoAddItem. When the item is created, Inventory_X also creates the child containers defined by the item.

Debug Add
Inventory.TryAutoAddItem(DA_Item_Backpack, BackpackContainer, 1)

Reading Child Containers

When you select an item in the UI:

Open Nested Container
On Item Double Click
  ItemView = selected item view
  If ItemView.bHasChildContainers:
    ChildViews = Inventory.GetChildContainerViews(ItemView.ItemHandle)
    ForEach ChildView:
      Create WBP_ContainerGrid
      SetContainerView(ChildView)

Recommended UI

There are two valid approaches:

  • Open a separate panel when the player inspects the backpack.
  • Show child containers inline under the tooltip or in a details panel.

For tutorials and debugging, the separate panel is clearer.

Safety Rules

Inventory_X validates nesting and protects against cycles and excessive depth. This means you should not try to move a container into itself through creative UI logic.

ResultWhen it happens
CycleDetectedAn operation would create a container loop.
MaxNestingDepthExceededThe maximum nesting depth would be exceeded.
RuleDeniedThe container or item does not allow that nesting case.

Verification

  • After adding DA_Item_Backpack, bHasChildContainers in the ItemView is true.
  • GetChildContainerViews returns at least one container.
  • You can draw the child container using the same WBP_ContainerGrid already created.
