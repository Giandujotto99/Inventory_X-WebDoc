Layer 1 - Building A Basic Inventory UI

Inventory_X does not impose a UI. In this tutorial, you create a Blueprint UI that reads component views and displays grids, equipment slots, icons, quantity, and tooltips.

Widgets To Create

WidgetResponsibility
WBP_InventoryRootMain widget. Receives the player inventory reference and coordinates refresh.
WBP_ContainerGridDraws a grid-based container from FInventoryContainerView.
WBP_InventoryCellSingle grid cell. Stores X/Y coordinates and container handle.
WBP_ItemIconRepresents an item. Stores FInventoryItemView and displays icon, quantity, and size.
WBP_EquipmentPanelDraws a slot-based container.
WBP_ItemTooltipDisplays display name, description, fragments, and debug info.

Root Widget

In WBP_InventoryRoot, create these variables:

  • InventoryComponent: type InventoryComponent Object Reference, Expose on Spawn.
  • BackpackGridWidget: reference to the grid widget.
  • EquipmentWidget: reference to the equipment widget.
Refresh Flow
Function RefreshInventory
  Is Valid InventoryComponent?
  GetRootContainerViews
  ForEach ContainerView:
    Branch bIsGridBased -> BackpackGridWidget.SetContainerView(ContainerView)
    Branch bIsSlotBased -> EquipmentWidget.SetContainerView(ContainerView)

Opening The UI

In the player character or player controller, create an input to open and close the UI.

Blueprint Flow
Input ToggleInventory
  If InventoryWidget is not valid:
    Create Widget WBP_InventoryRoot
      InventoryComponent = Character.Inventory
    Add To Viewport
    Bind Inventory.OnInventoryChanged -> InventoryWidget.RefreshInventory
    RefreshInventory
  Else:
    Remove From Parent
    Clear reference

Drawing A Grid

WBP_ContainerGrid should receive an FInventoryContainerView. Use Dimensions.X and Dimensions.Y to create cells.

  1. Clear the cell panel.
  2. Loop Y from 0 to Dimensions.Y - 1.
  3. Loop X from 0 to Dimensions.X - 1.
  4. Create WBP_InventoryCell and save coordinates plus container handle.
  5. After creating the cells, loop through the container Items and create a WBP_ItemIcon for each item.

Loading Cosmetics

To get display names and icons, use the Inventory_XClient module.

Item Icon
WBP_ItemIcon.SetItemView(ItemView)
  Get ItemDefinition from ItemView
  AsyncLoadItemCosmetic
    OnLoaded:
      Set Display Name
      Async Load Icon texture if assigned
      Set Brush From Texture
  Show Quantity if ItemView.Quantity > 1
Why use AsyncLoadItemCosmetic

LoadItemCosmetic blocks until the asset is loaded. It is fine for debugging, but in production UI prefer AsyncLoadItemCosmetic.

Tooltip

When the mouse hovers over WBP_ItemIcon, create WBP_ItemTooltip and pass the ItemView. In the tooltip, show:

  • Display name and description from the cosmetic.
  • Quantity.
  • Effective size.
  • Runtime tags.
  • Available fragments, using GetItemFragments or the TryGet... nodes.

Verification

  • Opening the UI shows backpack and equipment.
  • The items added in the previous tutorial are visible.
  • Stack quantities are correct.
  • When you add or remove items, OnInventoryChanged refreshes the UI.
